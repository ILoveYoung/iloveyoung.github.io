Шаг 1. Регистрируемся на GitHub Для этого перейдите на главную страницу и нажмите Sign up в левом верхнем углу. Либо же можете воспользоваться прямой ссылкой. Шаг 2. Создаём репозиторий После того, как ваш аккаунт будет создан, вы окажетесь на главной странице. Нажмите кнопку Create repository. Либо же можете воспользоваться прямой ссылкой. Под словом Owner будет написано ваше имя пользователя, которое вы указали при регистрации. Если в вашем имени пользователя есть большие буквы, делаем их маленькими, после чего к имени пользователя добавляем .github.io и вставляем получившуюся конструкцию в поле Repository name. Например, если ваше имя пользователя TestNickName111, то в поле Repository name вы должны написать следующий текст: testnickname111.github.io Больше на этой странице ничего делать не нужно. Листаем вниз и нажимаем Create repository. Шаг 3. Добавляем WTFPL license После создания репозитория, вы оказываетесь на его главной странице. Листните немного вниз. Нам нужен следующий текст: Нажмите на LICENSE. Вы окажетесь на следующей странице:

Переходим на официальную страницу лицензии, копируем текст, возвращаемся на ГитХаб и вставляем скопированный текст в поле Enter file contents here. После этого нажимаем Commit changes... И подтверждаем. После этого переходим на главную страницу репозитория. Вы всегда сможете найти свои репозитории в следующих местах:
  • Раздел Repositories вверху главной страницы ГитХаба
  • Раздел Repositories внизу меню (кнопка в левом верхнем углу)
  • Специальная отдельная страница, которая в нашем примере будет находиться по следующей ссылке: https://github.com/TestNickName111?tab=repositories
Главная страница репозитория в нашем примере находится по следующей ссылке: https://github.com/TestNickName111/testnickname111.github.io Шаг 4. Создаём главную страницу сайта Нажимаем на Нажимаем Create new file. В поле Name your file... напишем index.html (это обязательно именно так), а в поле Enter file contents here напишем Версия 1 (это текст просто для примера, он необязателен). После чего сохраняем файл по уже знакомой схеме (2 раза нажать Commit changes). И возвращаемся на главную страницу репозитория. Главная страница вашего сайта будет находиться по следующей ссылке: https://testnickname111.github.io Шаг 5. Последние штрихи Нажимаем на Нажимаем Settings. Найдите пункты Wikis, Issues и Projects. И уберите у них галочку. Далее возвращаемся на главную страницу репозитория. Включаем в браузере режим «Версия для ПК» и нажимаем на Откроется вот такое меню: Ставим галочку напротив Use your GitHub Pages website, а у пунктов Releases, Packages и Deployments галочку снимаем. Нажимаем Save changes. И отключаем режим ПК. Если отобразилась ошибка 404, проверьте, что вы находитесь именно на главной странице репозитория (см. конец раздела «Шаг 3») Как загрузить файл Возникает вопрос: «Окей, текстовые файлы можно создавать прямо в репозитории. А что если у меня есть изображение? Как я могу загрузить его на сервер?»

Ответ: нажимаем на Нажимаем Upload files. Нажимаем choose your files. После этого выбираем один или несколько файлов (они могут быть и текстовыми), листаем вниз и нажимаем Commit changes. Чтобы просмотреть загруженный файл, нажмите View code. И выберите нужный файл. Чтобы вернуться назад, как обычно нажимаем на название репозитория. Как создать папку Ладно, идём дальше. Как создавать и загружать файлы — это понятно. Но что если у меня их много? Как я могу их сгруппировать?
  1. Переходим в окно создания нового файла ( → Create new file)
  2. Пишем в поле Name your file... название папки
  3. Вводим / Как видим, после того, как мы ввели слэш, название папки было вынесено за пределы поля ввода, а кнопка Commit changes погасла
  4. Пишем название файла
  5. Сохраняем
Более правильно это называется не папка (англ. folder), а директория (англ. directory).

Так вот, различие в том, что если в файловом менеджере вы можете создать пустую папку, то вот директория пустой быть не может — в ней должен быть хотя бы 1 файл. Поэтому, если вы хотите загрузить, например, картинку в новую директорию, вам сначала надо создать в этой директории текстовый файл, а потом уже загружать картинку (или любые другие файлы). Соответственно, если созданный текстовый файл вам не нужен, вы можете его удалить. Как это сделать, описано ниже. Как удалить Открываем страницу просмотра файла и нажимаем на Нажимаем Delete file Нажимаем Commit changes... И подтверждаем. Как редактировать файл Непосредственно в самом ГитХабе вы имеете возможность редактировать только текстовые файлы (в том числе и svg). Для этого выберите необходимый файл и нажмите Нажимаем Edit in place. Откроется окно редактирования файла. Введите свой текст. Нажмите на кнопку Preview, чтобы открыть окно предпросмотра (предварительного просмотра), где вы можете сравнить версии файла до редактирования и после. Нажимаем Commit changes... И подтверждаем.
